快速開始

本指南將帶您完成 IPRO SDK 的安裝設置，並建構您的第一個應用程式。

系統需求

開始之前，請確保您的開發環境符合以下需求：

硬體需求

  • IPRO7 開發板（EVB 或 FPGA 版本）
  • JTAG 除錯器（用於燒錄與除錯）
  • USB 連接線

軟體需求

  • 作業系統：Linux（Ubuntu 20.04+ 建議）
  • 工具鏈：RISC-V GCC 交叉編譯器
  • 建構工具：CMake 3.20+, Ninja
  • Python：Python 3.8+（用於 west 與腳本）
  • Git：版本控制
安裝基本工具（Ubuntu/Debian） 
# 安裝建構工具
sudo apt update
sudo apt install -y cmake ninja-build python3 python3-pip git

# 驗證安裝
cmake --version
ninja --version
python3 --version

安裝設置

1. 取得 SDK 原始碼

📋 注意：IPRO SDK 原始碼託管於內部 GitLab，請聯繫管理員取得存取權限。
Clone SDK（內部 GitLab） 
# 從內部 GitLab Clone IPRO SDK
# 請替換為您的 GitLab 伺服器位址
git clone https://your-gitlab-server/ipro/ipro_sdk.git
cd ipro_sdk

2. 設置工具鏈

IPRO SDK 使用 RISC-V 交叉編譯器。預設路徑為 /opt/toolchain/riscv_ipro7/。

驗證工具鏈 
# 確認工具鏈存在
ls /opt/toolchain/riscv_ipro7/bin/

# 應該看到類似這些檔案：
# riscv64-unknown-elf-gcc
# riscv64-unknown-elf-g++
# riscv64-unknown-elf-objcopy
# ...
⚠️ 注意：如果您的工具鏈位於不同路徑，請在建構時指定 CROSS_COMPILE 參數。

Zephyr RTOS 快速開始

Zephyr 是推薦的開發方式，提供現代化的建構系統和豐富的功能。

一鍵設置環境

Bootstrap Zephyr 
# 執行 bootstrap 腳本
./tools/zephyr/bootstrap.sh

# 腳本會自動：
# - 下載並安裝 Zephyr SDK
# - 安裝 west 建構工具
# - 設置環境變數
# - 下載必要的 Zephyr 模組

使用互動式選單

互動式開發 
# 啟動互動式選單（支援中英雙語）
./tools/zephyr/menu.sh

直接建構應用

建構 Zephyr 應用 
# 使用增強建構腳本
./tools/zephyr/build.sh -a ipro_zephyr_demo -b ipro7_fpga -c

# 或使用 west 命令
cd zephyr-workspace
source setup_env.sh
west build -b ipro7_fpga ../apps/ipro_zephyr_demo

FreeRTOS 快速開始

傳統的 FreeRTOS 開發方式，適合熟悉 FreeRTOS 的開發者。

使用簡化建構腳本

FreeRTOS 建構 
# 建構 BLE Demo 應用
./build_freertos.sh ipro_ble_demo

# 建構 IPRO7 Demo 應用
./build_freertos.sh ipro7_demo

# 建構 FPGA BLE 5.4 應用
./build_freertos.sh ipro7_fpga_ble_5p4

手動 CMake 建構

CMake 建構指令 
# 設定專案
cmake -S . -B build \
    -G "Ninja" \
    -DIPRO_SDK_BASE=$(pwd) \
    -DCONFIG_BUILD_PORJECT=ipro_ble_demo \
    -DCMAKE_PREFIX_PATH=$(pwd)/cmake \
    -DCUSTOM_CONFIG_DIR=$(pwd)/apps/ipro_ble_demo \
    -DCONFIG_APPS=1 \
    -DCROSS_COMPILE=/opt/toolchain/riscv_ipro7/bin/riscv64-unknown-elf-

# 執行建構
cmake --build build

第一次建構

讓我們建構並驗證 BLE Demo 應用。

步驟 1：建構應用

建構 BLE Demo 
./build_freertos.sh ipro_ble_demo

步驟 2：檢查建構輸出

檢查輸出檔案 
# 列出建構產出
ls -la apps/ipro_ble_demo/build/build_out/

# 應該看到：
# ipro_ble_demo.elf  - ELF 執行檔
# ipro_ble_demo.bin  - 二進位燒錄檔
# ipro_ble_demo.map  - 記憶體映射檔

步驟 3：燒錄到開發板

使用 OpenOCD 燒錄 
# 使用 OpenOCD 燒錄（需連接 JTAG）
openocd -f apps/ipro_ble_demo/debug.cfg \
    -c "program apps/ipro_ble_demo/build/build_out/ipro_ble_demo.elf verify reset exit"
✅ 成功！如果建構完成且沒有錯誤，您已經成功設置 IPRO SDK 開發環境。

除錯設置

使用 GDB + OpenOCD

啟動除錯 
# 終端機 1：啟動 OpenOCD
openocd -f apps/ipro_ble_demo/debug.cfg

# 終端機 2：啟動 GDB
/opt/toolchain/riscv_ipro7/bin/riscv64-unknown-elf-gdb \
    apps/ipro_ble_demo/build/build_out/ipro_ble_demo.elf \
    -ex "target remote :3333" \
    -ex "monitor reset halt"

VS Code 除錯配置

SDK 提供 VS Code 擴充套件，支援圖形化除錯。詳細設定請參考 tools/vscode_extension/。
