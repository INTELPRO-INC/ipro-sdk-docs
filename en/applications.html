ipro7_demo

Comprehensive platform demonstration application showcasing complete SDK functionality including multimedia, USB, and peripheral drivers.

Features

  • Peripheral Drivers: GPIO, UART, SPI (Master/Slave), I2C, PWM, ADC, DAC
  • USB Device: CDC, MSC, HID, UAC (Audio), UVC (Video) via CherryUSB
  • Multimedia: ISP Camera, MJPEG Codec, LCD Display (ILI9488)
  • Audio: AUADC Audio Input, DAC Speaker Output
  • Network: EMAC Ethernet, lwIP Protocol Stack, iPerf Testing
  • Storage: SD Card (SDH), FatFS File System
  • FreeRTOS: Multi-task Management & Runtime Statistics

Build

Build Platform Demo 
make -C apps/ipro7_demo

Code Example

GPIO Usage Example 
#include "hal_gpio.h"

void gpio_example(void)
{
    // Configure GPIO as output
    gpio_set_mode(GPIO_PIN_0, GPIO_OUTPUT_MODE);

    // Set output high
    gpio_write(GPIO_PIN_0, 1);

    // Read input
    gpio_set_mode(GPIO_PIN_1, GPIO_INPUT_PP_MODE);
    int value = gpio_read(GPIO_PIN_1);

    // Toggle output
    gpio_toggle(GPIO_PIN_0);
}

ipro_unit_test

SDK unit test framework with automated build, flash, and test result parsing support.

Features

  • Auto-run: CONFIG_TEST_AUTO_RUN=y runs tests on boot
  • Automation Script: tools/board_test.sh supports full build+flash+monitor+parse workflow
  • Result Detection: Outputs ALL TESTS PASSED or SOME TESTS FAILED

Build and Test

Full Test Workflow 
# Full cycle: build + flash + capture serial test results
tools/board_test.sh --project apps/ipro_unit_test

# Quick retest (reboot board, no rebuild/reflash)
python3 tools/serial_monitor.py /dev/cu.usbserial-2120 --send reboot --timeout 30 \
    --wait-for "ALL TESTS PASSED|SOME TESTS FAILED"

ipro_throughput_test

High-performance BLE throughput test application using native Zephyr bt_* APIs, achieving 999 Kbps between two IPRO7 development boards.

Features

  • Native Zephyr APIs: Direct use of bt_gatt_notify/bt_gatt_write_without_response
  • Bidirectional Data Transfer: Custom GATT service
  • Real-time Statistics: Live Kbps throughput display
  • PHY Selection: 1M/2M/Coded
  • DLE Optimization: Data Length Extension 251 bytes
  • TX Descriptor Optimization: Multiple TX descriptors for higher throughput

Performance Results

Configuration Throughput
1M PHY, 7.5ms CI, 244B ~180-200 Kbps
2M PHY, 7.5ms CI, 244B ~350-400 Kbps
2M PHY + DLE 251B + TX Desc 999 Kbps

Quick Start

Test Workflow 
# Board 1 (Slave)
tp slave

# Board 2 (Master)
tp master

# Start test after connection (244 bytes, 10 seconds)
tp start 244 10

# Optimize speed
tp phy 2         # Use 2M PHY
tp conn 6 6      # Minimum connection interval

Build

Build Throughput Test 
make -C apps/bluetooth/ipro_throughput_test

ipro_ble_remote

Interactive BLE remote control application designed for mobile app development with 6 GATT services.

Features

  • 6 GATT Services: LED Control, Button, Environment, UART, PIR Image, OTA Update
  • Mobile Friendly: Complete protocol spec for Android/iOS app development
  • Image Transfer: 320x480 JPEG via BLE with MTU 517 support
  • High-speed OTA: MTU 517 auto-negotiation, 12.3 KB/s transfer speed

Service List

Service UUID Function
LED Control 0xFF10 On/Off, Brightness (0-100%), Blink Mode
Button 0xFF20 Status Notification, Press Count
Environment 0xFF30 Temperature, Humidity (Simulated)
UART 0xFF40 Bidirectional Text Communication
PIR Image 0xFF50 Motion Detection + JPEG Transfer
OTA 0xFF60 Firmware Update (800KB max, 12.3 KB/s)

Creating New Applications

Follow these steps to create your own application.

Step 1: Create Directory

Create Application Directory 
mkdir -p apps/my_app/include

Step 2: Create CMakeLists.txt

apps/my_app/CMakeLists.txt 
cmake_minimum_required(VERSION 3.13)

# Set application name
set(APP_NAME my_app)

# Include SDK build system
include(${IPRO_SDK_BASE}/cmake/extension.cmake)

# Add source files
target_sources(app PRIVATE
    main.c
)

# Add include directories
target_include_directories(app PRIVATE
    include
)

Step 3: Create Main Program

apps/my_app/main.c 
#include <stdio.h>
#include "FreeRTOS.h"
#include "task.h"
#include "ipro_log.h"

#define TAG "MY_APP"

void app_task(void *params)
{
    IPRO_LOG_INFO(TAG, "My App Started!");

    while (1) {
        IPRO_LOG_INFO(TAG, "Running...");
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

int main(void)
{
    // System initialization
    system_init();

    // Create application task
    xTaskCreate(app_task, "app", 4096, NULL, 5, NULL);

    // Start scheduler
    vTaskStartScheduler();

    return 0;
}

Step 4: Build

Build New Application 
make -C apps/my_app
