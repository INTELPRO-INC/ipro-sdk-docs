Getting Started

This guide will walk you through setting up the IPRO SDK and building your first application.

Prerequisites

Before starting, ensure your development environment meets the following requirements:

Hardware Requirements

  • IPRO7 Development Board (EVB or FPGA version)
  • JTAG Debugger (for flashing and debugging)
  • USB Cable

Software Requirements

  • Operating System: Linux (Ubuntu 20.04+ recommended)
  • Toolchain: RISC-V GCC Cross Compiler
  • Build Tools: CMake 3.20+, Ninja
  • Python: Python 3.8+ (for west and scripts)
  • Git: Version control
Install Basic Tools (Ubuntu/Debian) 
# Install build tools
sudo apt update
sudo apt install -y cmake ninja-build python3 python3-pip git

# Verify installation
cmake --version
ninja --version
python3 --version

2. Setup Toolchain

IPRO SDK uses RISC-V cross compiler. Default path is /opt/toolchain/riscv_ipro7/.

Verify Toolchain 
# Verify toolchain exists
ls /opt/toolchain/riscv_ipro7/bin/

# You should see files like:
# riscv64-unknown-elf-gcc
# riscv64-unknown-elf-g++

Zephyr RTOS Quick Start

Zephyr is the recommended development approach, offering a modern build system and rich features.

One-Click Environment Setup

Bootstrap Zephyr 
# Run bootstrap script
./tools/zephyr/bootstrap.sh

# The script will automatically:
# - Download and install Zephyr SDK
# - Install west build tool
# - Setup environment variables
# - Download required Zephyr modules

Use Interactive Menu

Interactive Development 
# Launch interactive menu (supports Chinese/English)
./tools/zephyr/menu.sh

Direct Build

Build Zephyr Application 
# Use enhanced build script
./tools/zephyr/build.sh -a ipro_zephyr_demo -b ipro7_fpga -c

# Or use west command
cd zephyr-workspace
source setup_env.sh
west build -b ipro7_fpga ../apps/ipro_zephyr_demo

FreeRTOS Quick Start

Traditional FreeRTOS development approach, suitable for developers familiar with FreeRTOS.

Use Simplified Build Script

FreeRTOS Build 
# Build BLE Demo application
./build_freertos.sh ipro_ble_demo

# Build IPRO7 Demo application
./build_freertos.sh ipro7_demo

# Build FPGA BLE 5.4 application
./build_freertos.sh ipro7_fpga_ble_5p4

Manual CMake Build

CMake Build Commands 
# Configure project
cmake -S . -B build \
    -G "Ninja" \
    -DIPRO_SDK_BASE=$(pwd) \
    -DCONFIG_BUILD_PORJECT=ipro_ble_demo \
    -DCMAKE_PREFIX_PATH=$(pwd)/cmake \
    -DCUSTOM_CONFIG_DIR=$(pwd)/apps/ipro_ble_demo \
    -DCONFIG_APPS=1 \
    -DCROSS_COMPILE=/opt/toolchain/riscv_ipro7/bin/riscv64-unknown-elf-

# Execute build
cmake --build build

First Build

Let's build and verify the BLE Demo application.

Step 1: Build Application

Build BLE Demo 
./build_freertos.sh ipro_ble_demo

Step 2: Check Build Output

Check Output Files 
# List build artifacts
ls -la apps/ipro_ble_demo/build/build_out/

# You should see:
# ipro_ble_demo.elf  - ELF executable
# ipro_ble_demo.bin  - Binary flash file
# ipro_ble_demo.map  - Memory map file

Step 3: Flash to Development Board

Flash with OpenOCD 
# Flash using OpenOCD (requires JTAG connection)
openocd -f apps/ipro_ble_demo/debug.cfg \
    -c "program apps/ipro_ble_demo/build/build_out/ipro_ble_demo.elf verify reset exit"
‚úÖ Success! If the build completes without errors, you have successfully set up the IPRO SDK development environment.

Debugging Setup

Using GDB + OpenOCD

Start Debugging 
# Terminal 1: Start OpenOCD
openocd -f apps/ipro_ble_demo/debug.cfg

# Terminal 2: Start GDB
/opt/toolchain/riscv_ipro7/bin/riscv64-unknown-elf-gdb \
    apps/ipro_ble_demo/build/build_out/ipro_ble_demo.elf \
    -ex "target remote :3333" \
    -ex "monitor reset halt"

VS Code Debug Configuration

The SDK provides VS Code extension for graphical debugging. See tools/vscode_extension/ for detailed setup.
