BLE Development Guide

This guide explains how to develop Bluetooth Low Energy applications using the IPRO SDK's Zephyr BLE Host Stack. The SDK uses standard Zephyr bt_* APIs with a RivieraWaves controller, providing full BLE 5.4 functionality.

Core Concepts

  • Zephyr bt_* API: Applications use standard Zephyr BLE APIs (replacing deprecated ipro_ble_api.h)
  • Callback-driven: Register connection callbacks with bt_conn_cb_register()
  • Static service definition: Define GATT services using BT_GATT_SERVICE_DEFINE() macro
  • Unified connection pool: In linkN variants, N = total connections, any Central/Peripheral mix

Controller Variants (linkN Naming)

The controller uses a unified connection pool architecture. In linkN, N represents the total simultaneous connection count. Each connection slot can be used as Central or Peripheral:

Variant Connections TX Buffers Role Support Description
link1_periph 1 2 Peripheral only Minimal footprint, peripheral only
link1 1 2 All roles Single connection, all roles
link2 2 4 All roles Dual connection (2C+0P, 1C+1P, 0C+2P)
link4 4 6 All roles Four connections, any combination
link8 8 8 All roles Eight connections, any combination

LE Audio and BLE 5.3/5.4 Suffixes

Base variants can have suffixes appended to enable advanced features:

  • _leaudio ‚Äî BLE 5.2 + LE Audio (BAP, CAP, VCP, MCP, CSIP)
  • _leaudio_53_54 ‚Äî BLE 5.4 + LE Audio (most complete feature set)

Examples: link1_leaudio, link2_leaudio_53_54

Getting Started

1. Initialize BLE Subsystem

BLE Initialization (Zephyr bt_* API) 
#include <bluetooth/bluetooth.h>
#include <bluetooth/conn.h>
#include <bluetooth/gatt.h>

/* Connection callbacks */
static void connected(struct bt_conn *conn, u8_t err)
{
    if (err) {
        printk("Connection failed (err %u)\n", err);
        return;
    }
    printk("Connected\n");
}

static void disconnected(struct bt_conn *conn, u8_t reason)
{
    printk("Disconnected (reason %u)\n", reason);
}

static struct bt_conn_cb conn_callbacks = {
    .connected = connected,
    .disconnected = disconnected,
};

/* BLE ready callback */
static void bt_ready(int err)
{
    if (err) {
        printk("Bluetooth init failed (err %d)\n", err);
        return;
    }
    printk("Bluetooth initialized\n");

    /* Start advertising */
    start_advertising();
}

void app_init(void)
{
    /* Register connection callbacks */
    bt_conn_cb_register(&conn_callbacks);

    /* Enable Bluetooth */
    int err = bt_enable(bt_ready);
    if (err) {
        printk("bt_enable failed (err %d)\n", err);
    }
}

2. Configure Advertising

Advertising Setup 
static const struct bt_data ad[] = {
    BT_DATA_BYTES(BT_DATA_FLAGS,
                  BT_LE_AD_GENERAL | BT_LE_AD_NO_BREDR),
    BT_DATA_BYTES(BT_DATA_UUID16_ALL,
                  0x0d, 0x18),  /* Heart Rate Service */
};

static const struct bt_data sd[] = {
    BT_DATA(BT_DATA_NAME_COMPLETE, "IPRO Device", 11),
};

void start_advertising(void)
{
    int err = bt_le_adv_start(BT_LE_ADV_CONN_NAME,
                              ad, ARRAY_SIZE(ad),
                              sd, ARRAY_SIZE(sd));
    if (err) {
        printk("Advertising failed to start (err %d)\n", err);
    }
}

GATT Services

Static Service Definition

Custom GATT Service Example 
#include <bluetooth/gatt.h>
#include <bluetooth/uuid.h>

/* Define custom UUIDs */
static struct bt_uuid_128 custom_svc_uuid = BT_UUID_INIT_128(
    0xf0, 0xde, 0xbc, 0x9a, 0x78, 0x56, 0x34, 0x12,
    0x78, 0x56, 0x34, 0x12, 0x78, 0x56, 0x34, 0x12);

static struct bt_uuid_128 custom_char_uuid = BT_UUID_INIT_128(
    0xf1, 0xde, 0xbc, 0x9a, 0x78, 0x56, 0x34, 0x12,
    0x78, 0x56, 0x34, 0x12, 0x78, 0x56, 0x34, 0x12);

static uint8_t custom_value[512];

/* Read callback */
static ssize_t read_custom(struct bt_conn *conn,
                           const struct bt_gatt_attr *attr,
                           void *buf, u16_t len, u16_t offset)
{
    return bt_gatt_attr_read(conn, attr, buf, len, offset,
                             custom_value, sizeof(custom_value));
}

/* Statically define service using macro */
BT_GATT_SERVICE_DEFINE(custom_svc,
    BT_GATT_PRIMARY_SERVICE(&custom_svc_uuid),
    BT_GATT_CHARACTERISTIC(&custom_char_uuid.uuid,
                           BT_GATT_CHRC_READ | BT_GATT_CHRC_NOTIFY,
                           BT_GATT_PERM_READ,
                           read_custom, NULL, custom_value),
    BT_GATT_CCC(NULL, BT_GATT_PERM_READ | BT_GATT_PERM_WRITE),
);

/* Send notification */
void notify_custom_value(struct bt_conn *conn)
{
    bt_gatt_notify(conn, &custom_svc.attrs[1],
                   custom_value, sizeof(custom_value));
}

LE Audio

The SDK supports complete LE Audio functionality, requiring controller variants with the _leaudio suffix. Headers are located at bluetooth/audio/.

Supported LE Audio Profiles

  • BAP ‚Äî Basic Audio Profile (unicast/broadcast audio streams)
  • CAP ‚Äî Common Audio Profile (coordinated audio operations)
  • VCP ‚Äî Volume Control Profile
  • MCP ‚Äî Media Control Profile
  • CSIP ‚Äî Coordinated Set Identification Profile

LE Audio Services

  • ASCS (0x184E) - Audio Stream Control Service
  • PACS (0x1850) - Published Audio Capabilities Service
  • CAS (0x1853) - Common Audio Service

Throughput Optimization

Connection Parameters and TX Descriptors

Key parameters that affect throughput:

  • max_ce_len: Maximum connection event length; larger values allow more packets per event
  • BLE_NB_TX_DESC_PER_CON: TX descriptors per connection
  • CONFIG_BLE_TX_BUFF_DATA: TX buffer count (auto-scaled by linkN variant)

MTU Optimization

MTU and High-Speed Transfer 
/* Calculate optimal payload after MTU negotiation */
uint16_t mtu = bt_gatt_get_mtu(conn);
uint16_t payload = (mtu > 3) ? (mtu - 3) : 20;  /* ATT header = 3 bytes */

/* Send notification */
bt_gatt_notify(conn, attr, data, payload);

  • MTU 517 + optimized TX descriptors: 999 Kbps measured throughput
  • Image transfer: 56KB in 542ms, achieving 101.4 KB/s (811 Kbps)
  • TX buffers auto-scale by variant: link1=2, link2=4, link4=6, link8=8
