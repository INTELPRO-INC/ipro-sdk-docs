應用範例

ipro_ble_demo ⭐

最完整的 BLE 展示應用，推薦作為 BLE 開發的起點。

功能特色

  • Core/Profile 架構：清晰的層次分離，易於擴展
  • 多服務支援：Heart Rate, Battery, UART, Device Info
  • LE Audio：GAF 框架整合，LC3 編解碼支援
  • 完整 CLI：40+ 互動指令用於測試與除錯
  • 自動重連：斷線後自動恢復廣播
  • OTA 更新：支援無線韌體更新（MTU 517，12.3 KB/s）
  • MTU 優化：動態 MTU 適配，最高支援 517 bytes

目錄結構

apps/ipro_ble_demo/
├── main.c                    # 應用入口
├── ipro_ble_demo_app.c       # BLE 應用邏輯

CLI 指令範例

常用 CLI 指令 
# 系統資訊
help                  # 顯示所有指令
ble status            # BLE 狀態

# 廣播控制
ble adv start         # 開始廣播
ble adv stop          # 停止廣播
ble adv data          # 顯示廣播資料

# 連線管理
ble conn list         # 列出連線
ble conn info 0       # 連線詳情
ble conn disconnect 0 # 斷開連線

# 服務操作
ble hrs update 80     # 更新心率
ble bas update 75     # 更新電量
ble uart send "Hello" # 發送 UART 資料

CONFIG_IPRO_BLE_DIS_ENABLE=y
CONFIG_IPRO_BLE_UART_ENABLE=y

ipro_ble_remote 🎮 NEW

互動式 BLE 遙控器應用，專為手機 App 開發設計。

功能特色

  • 6 個 GATT 服務：LED 控制、按鈕、環境感測、UART、PIR 圖片、OTA 更新
  • 手機友好：完整的協定規格，適合 Android/iOS App 開發
  • 圖片傳輸：320×480 JPEG 透過 BLE 傳輸，MTU 517 支援
  • 高速 OTA：MTU 517 自動協商，12.3 KB/s 傳輸速度
  • 影像傳輸：40 KB/s (320 Kbps) 圖片傳輸效能
  • 記憶體優化：390KB flash (37.24%)，零警告

服務列表

服務 UUID 功能
LED Control 0xFF10 開關、亮度 (0-100%)、閃爍模式
Button 0xFF20 狀態通知、按壓計數
Environment 0xFF30 溫度、濕度（模擬）
UART 0xFF40 雙向文字通訊
PIR Image 0xFF50 動態偵測 + JPEG 傳輸 (101 KB/s ⚡)
OTA 0xFF60 韌體更新 (800KB max, 12.3 KB/s)

OTA 效能規格

MTU 負載大小 傳輸速度 250KB 韌體時間
513+ 504 bytes 12.3 KB/s ~20 秒
247 244 bytes 2.0 KB/s ~125 秒
💡 建議：手機 App 連線後使用 requestMtu(513) 以獲得最佳效能。

# 查看詳細文件
cat apps/ipro_ble_remote/README.md

ipro_throughput_test 📊

高效能 BLE 傳輸測試應用，用於兩個 IPRO7 開發板之間的效能測試。

功能特色

  • 雙向資料傳輸：使用自訂 GATT 服務
  • 即時統計：顯示 Kbps 傳輸速度
  • 可調封包大小：20-244 bytes
  • PHY 選擇：1M/2M/Coded
  • 連線參數調整：優化傳輸效能

效能測試結果

配置 傳輸速度
1M PHY, 7.5ms CI, 244B ~180-200 Kbps
2M PHY, 7.5ms CI, 244B ~350-400 Kbps
2M PHY + DLE 優化 600+ Kbps

快速開始

測試流程 
# 開發板 1 (Slave)
tp slave

# 開發板 2 (Master)
tp master

# 連線後開始測試（244 bytes, 10 秒）
tp start 244 10

# 優化速度
tp phy 2         # 使用 2M PHY
tp conn 6 6      # 最小連線間隔

建構與執行

建構 Throughput Test 
./build_freertos.sh ipro_throughput_test build

GPIO 使用範例 
#include "ipro_hal_gpio.h"

void gpio_example(void)
{
    // 配置 GPIO 為輸出
    ipro_gpio_config_t config = {
        .mode = IPRO_GPIO_MODE_OUTPUT,
        .pull = IPRO_GPIO_PULL_NONE,
    };
    ipro_hal_gpio_init(GPIO_PIN_0, &config);
    
    // 設置輸出
    ipro_hal_gpio_write(GPIO_PIN_0, 1);
    
    // 讀取輸入
    uint8_t value = ipro_hal_gpio_read(GPIO_PIN_1);
}

建立新應用

按照以下步驟建立您自己的應用程式。

步驟 1：建立目錄

建立應用目錄 
mkdir -p apps/my_app/include

步驟 2：建立 CMakeLists.txt

apps/my_app/CMakeLists.txt 
cmake_minimum_required(VERSION 3.13)

# 設置應用名稱
set(APP_NAME my_app)

# 包含 SDK 建構系統
include(${IPRO_SDK_BASE}/cmake/extension.cmake)

# 添加原始檔
target_sources(app PRIVATE
    main.c
)

# 添加頭文件目錄
target_include_directories(app PRIVATE
    include
)

步驟 3：建立 proj.conf

apps/my_app/proj.conf 
# 基本配置
CONFIG_LOG_DEFAULT_LEVEL=3

# 根據需要啟用元件
# CONFIG_IPRO_BLE_LIB_ENABLE=y
# CONFIG_LWIP_ENABLE=y

步驟 4：建立主程式

apps/my_app/main.c 
#include <stdio.h>
#include "FreeRTOS.h"
#include "task.h"
#include "ipro_log.h"

#define TAG "MY_APP"

void app_task(void *params)
{
    IPRO_LOG_INFO(TAG, "My App Started!");
    
    while (1) {
        IPRO_LOG_INFO(TAG, "Running...");
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

int main(void)
{
    // 系統初始化
    system_init();
    
    // 建立應用任務
    xTaskCreate(app_task, "app", 4096, NULL, 5, NULL);
    
    // 啟動排程器
    vTaskStartScheduler();
    
    return 0;
}

步驟 5：建構

建構新應用 
./build_freertos.sh my_app
✅ 完成！您的新應用已建立，可以開始開發了。
