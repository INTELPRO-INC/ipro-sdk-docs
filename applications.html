應用範例

IPRO SDK Release 版本提供多個完整的參考應用，展示各種功能的最佳實踐。每個應用都可以直接建構、燒錄並測試。

應用概覽

本 Release SDK 包含 7 個參考應用，分為 BLE 應用（3 個）、LE Audio 應用（3 個）與平台應用（1 個）三大類：

ipro_ble_transparent apps/bluetooth/ble/ipro_ble_transparent FreeRTOS BLE 透傳通訊 串口透傳/資料橋接

ipro_ble_remote

互動式 BLE 遙控器應用，專為手機 App 開發設計，提供 6 個 GATT 服務。

功能特色

  • 6 個 GATT 服務：LED 控制、按鈕、環境感測、UART、PIR 圖片、OTA 更新
  • 手機友好：完整的協定規格，適合 Android/iOS App 開發
  • 圖片傳輸：320x480 JPEG 透過 BLE 傳輸，MTU 517 支援
  • 高速 OTA：MTU 517 自動協商，12.3 KB/s 傳輸速度

服務列表

服務 UUID 功能
LED Control 0xFF10 開關、亮度 (0-100%)、閃爍模式
Button 0xFF20 狀態通知、按壓計數
Environment 0xFF30 溫度、濕度（模擬）
UART 0xFF40 雙向文字通訊
PIR Image 0xFF50 動態偵測 + JPEG 傳輸
OTA 0xFF60 韌體更新 (800KB max, 12.3 KB/s)

建構

建構 BLE Remote 
make -C apps/bluetooth/ble/ipro_ble_remote

ipro_throughput_test

高效能 BLE 傳輸測試應用，使用原生 Zephyr bt_* API，在兩個 IPRO7 開發板之間達到 999 Kbps 傳輸速度。

功能特色

  • 原生 Zephyr API：直接使用 bt_gatt_notify/bt_gatt_write_without_response
  • 雙向資料傳輸：使用自訂 GATT 服務
  • 即時統計：顯示 Kbps 傳輸速度
  • PHY 選擇：1M/2M/Coded
  • DLE 最佳化：Data Length Extension 251 bytes
  • TX Descriptor 最佳化：多重傳輸描述符提升吞吐量

效能測試結果

配置 傳輸速度
1M PHY, 7.5ms CI, 244B ~180-200 Kbps
2M PHY, 7.5ms CI, 244B ~350-400 Kbps
2M PHY + DLE 251B + TX Desc 999 Kbps

快速開始

測試流程 
# 開發板 1 (Slave)
tp slave

# 開發板 2 (Master)
tp master

# 連線後開始測試（244 bytes, 10 秒）
tp start 244 10

# 最佳化速度
tp phy 2         # 使用 2M PHY
tp conn 6 6      # 最小連線間隔

建構

建構 Throughput Test 
make -C apps/bluetooth/ble/ipro_throughput_test

ipro_le_audio_headset

LE Audio 耳機參考應用，實作 GAF (Generic Audio Framework) 與 BAP (Basic Audio Profile)。

功能特色

  • GAF/BAP：完整的 LE Audio 通用音訊框架
  • LC3 Codec：低複雜度通訊編解碼
  • VCP：Volume Control Profile 音量控制
  • MCP：Media Control Profile 媒體控制
  • CAP：Common Audio Profile 公共音訊設定

建構

ipro7_demo

綜合平台展示應用，展示 SDK 完整功能，包含多媒體、USB 和週邊驅動。

功能特色

  • 週邊驅動：GPIO, UART, SPI (主/從), I2C, PWM, ADC, DAC
  • USB 裝置：CDC, MSC, HID, UAC (音訊), UVC (視訊) via CherryUSB
  • 多媒體：ISP 攝影機, MJPEG 編解碼, LCD 顯示 (ILI9488)
  • 音訊：AUADC 音訊輸入, DAC 喇叭輸出
  • 網路：EMAC 乙太網, lwIP 協定棧, iPerf 測試
  • 儲存：SD 卡 (SDH), FatFS 檔案系統
  • FreeRTOS：多任務管理與執行時間統計

建構

建構平台 Demo 
make -C apps/platform/ipro7_demo

程式碼範例

GPIO 使用範例 
#include "hal_gpio.h"

void gpio_example(void)
{
    // 配置 GPIO 為輸出
    gpio_set_mode(GPIO_PIN_0, GPIO_OUTPUT_MODE);

    // 設置輸出
    gpio_write(GPIO_PIN_0, 1);

    // 讀取輸入
    gpio_set_mode(GPIO_PIN_1, GPIO_INPUT_PP_MODE);
    int value = gpio_read(GPIO_PIN_1);

    // 切換輸出
    gpio_toggle(GPIO_PIN_0);
}

建立新應用

按照以下步驟建立您自己的應用程式。建議以 ipro_ble_remote 為範本複製後修改。

提示：最快的方式是複製現有應用作為起點：

cp -r apps/bluetooth/ble/ipro_ble_remote apps/bluetooth/ble/my_app

然後修改 CMakeLists.txt 中的 APP_NAME 和原始檔案即可。

步驟 1：建立目錄

應用需放在 apps/ 下的分類子目錄中，例如 BLE 應用放在 apps/bluetooth/ble/。

建立應用目錄 
mkdir -p apps/bluetooth/ble/my_app/include

步驟 2：建立 CMakeLists.txt

注意 IPRO_SDK_BASE 的相對路徑需根據應用所在的目錄深度調整。例如位於 apps/bluetooth/ble/my_app/ 時，需要向上四層才能到達 SDK 根目錄。

apps/bluetooth/ble/my_app/CMakeLists.txt 
cmake_minimum_required(VERSION 3.13)

# 設置應用名稱
set(APP_NAME my_app)

# 包含 SDK 建構系統（根據目錄深度調整相對路徑）
include(${IPRO_SDK_BASE}/cmake/extension.cmake)

# 添加原始檔
target_sources(app PRIVATE
    main.c
)

# 添加頭文件目錄
target_include_directories(app PRIVATE
    include
)

步驟 3：建立主程式

apps/bluetooth/ble/my_app/main.c 
#include <stdio.h>
#include "FreeRTOS.h"
#include "task.h"
#include "ipro_log.h"

#define TAG "MY_APP"

void app_task(void *params)
{
    IPRO_LOG_INFO(TAG, "My App Started!");

    while (1) {
        IPRO_LOG_INFO(TAG, "Running...");
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

int main(void)
{
    // 系統初始化
    system_init();

    // 建立應用任務
    xTaskCreate(app_task, "app", 4096, NULL, 5, NULL);

    // 啟動排程器
    vTaskStartScheduler();

    return 0;
}

步驟 4：建立 Makefile

Makefile 中的 IPRO_SDK_BASE 需要指向 SDK 根目錄。路徑深度取決於應用的位置。

apps/bluetooth/ble/my_app/Makefile 
# 根據應用目錄深度設定 SDK 根目錄相對路徑
# apps/bluetooth/ble/my_app/ -> 需向上 4 層
IPRO_SDK_BASE ?= ../../../..

include $(IPRO_SDK_BASE)/project.mk

步驟 5：建構

建構新應用 
make -C apps/bluetooth/ble/my_app
