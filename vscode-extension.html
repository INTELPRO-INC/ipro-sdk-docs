IPRO SDK VS Code Extension

專為 IPRO SDK 設計的 VS Code 擴充套件，支援本地與遠端開發環境，提供一鍵建構、燒錄等功能。

概覽

IPRO SDK Extension 是一套完整的開發工具套件，包含兩個互相協作的擴充套件：

🔧 IPRO SDK Extension (v0.0.7)

主要開發擴充套件，安裝於遠端伺服器或本地環境

  • 專案管理與配置
  • CMake 建構整合
  • 自動偵測工具鏈
  • 燒錄介面

⚡ IPRO Flash Tool Extension (v1.0.0)

燒錄工具擴充套件，安裝於本地 Windows 電腦

  • 本地燒錄執行
  • 即時進度回報
  • 自動偵測燒錄工具

支援的開發場景

場景 SDK 位置 燒錄工具 適用情況
本地開發 本地 Windows 本地 個人開發環境
遠端開發 遠端 Linux 伺服器 本地 Windows 團隊共用編譯伺服器

功能特色

🎯

智慧專案管理

自動偵測 SDK 內的所有應用專案，提供下拉選單快速切換。

🔧

自動工具鏈偵測

自動偵測 PATH 中的 cmake、ninja、riscv64-unknown-elf-gcc，無需手動配置。

🔨

一鍵建構

整合 CMake + Ninja 建構系統，提供 Configure 和 Build 按鈕。

⚡

遠端燒錄

透過 Base64 傳輸韌體到本地，支援即時進度顯示。

📊

即時進度

建構和燒錄過程中顯示即時進度條和詳細日誌。

🔗

Remote SSH 整合

完整支援 VS Code Remote SSH，遠端編譯本地燒錄。

安裝指南

遠端 Linux 伺服器準備

1. 安裝必要工具

安裝開發工具 
# 安裝 CMake 和 Ninja
sudo apt-get install cmake ninja-build

# 手動安裝 RISC-V 工具鏈（推薦）
# 下載並解壓工具鏈到 /opt/toolchain/riscv_ipro7/
echo 'export PATH=/opt/toolchain/riscv_ipro7/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 驗證安裝
which cmake ninja riscv64-unknown-elf-gcc
cmake --version
riscv64-unknown-elf-gcc --version

本地 Windows 準備

1. 安裝 VS Code 擴充功能

  1. 安裝 Visual Studio Code
  2. 安裝 Remote - SSH 擴充功能
  3. 準備燒錄工具 ipro_iot_tool.exe

2. 燒錄工具路徑

Extension 會自動偵測以下常見路徑：

常見燒錄工具路徑

C:\IPRO\tools\download\ipro_iot_tool.exe
C:\Program Files\IPRO\tools\download\ipro_iot_tool.exe

安裝 Extensions

⚠️ 重要安裝順序

  • IPRO SDK Extension：安裝在 遠端伺服器（透過 Remote SSH）
  • IPRO Flash Tool Extension：安裝在 本地（必須！）

安裝步驟

  1. 連接到遠端伺服器後，打開 Extensions 面板（Ctrl+Shift+X）
  2. 搜尋 IPRO SDK Extension，點擊 "Install in SSH: server-name"
  3. 中斷 SSH 連接或開啟新的本地視窗
  4. 搜尋 IPRO Flash Tool，點擊 "Install"（確認不是 "Install in SSH"）
  5. 重新載入視窗：F1 → Developer: Reload Window

開發流程

1. 連接遠端伺服器

  1. 按 F1 打開命令面板
  2. 選擇 Remote-SSH: Connect to Host...
  3. 選擇或新增 SSH 主機
  4. 輸入密碼或使用 SSH 金鑰

2. 打開 SDK 資料夾

  1. 選擇 File → Open Folder...
  2. 選擇 IPRO SDK 根目錄（例如：/home/username/ipro_sdk/）

3. 使用 IPRO 側邊欄

  1. 點擊左側活動列的 IPRO 圖示
  2. 在 Project Manager 選擇專案
  3. 點擊 Configure 按鈕進行配置
  4. 點擊 Build 按鈕進行編譯

遠端燒錄

燒錄步驟

1. 準備硬體

  1. 將開發板通過 USB 連接到本地 Windows 電腦
  2. 開啟裝置管理員，確認 COM 埠號（例如：COM4）

2. 開啟燒錄面板

  1. 在側邊欄 快速操作 區塊
  2. 點擊 Flash to Device 按鈕

3. 設定參數

參數 說明 預設值
COM Port 開發板連接的 COM 埠 COM4
Chip Name 晶片型號 ipro7
Baudrate 傳輸速率 2000000

4. 燒錄進度

進度 階段 說明
5% ✅ 連接裝置成功 與開發板建立連接
8% 📌 偵測到 Flash 晶片 識別 Flash 型號
25-30% 🔥 擦除主韌體區域 清除舊韌體
30-90% 📝 寫入主韌體中 燒錄新韌體
92% ✅ 主韌體寫入完成 燒錄完成
96% ✅ 主韌體驗證成功 校驗成功
100% 🎉 燒錄完成！ 全部完成

常見問題

Q1: 遠端無法找到 cmake 或工具鏈

問題：CMAKE_C_COMPILER not found in PATH

解決方法：

  1. 確認工具已安裝：which cmake riscv64-unknown-elf-gcc
  2. 添加到 PATH：echo 'export PATH=/opt/toolchain/riscv_ipro7/bin:$PATH' >> ~/.bashrc
  3. 重新載入視窗：F1 → Developer: Reload Window

Q2: 本地 Flash Tool Extension 未安裝

問題：無法呼叫本地 Flash Tool Extension

解決方法：

  1. 中斷 SSH 連接，在本地 VS Code 中安裝
  2. 確認顯示 "Install" 而非 "Install in SSH"
  3. 重新連接遠端並重新載入視窗

Q3: 燒錄工具路徑未配置

問題：Flash tool not configured

解決方法：

  1. 執行命令：F1 → IPRO: Configure Flash Tool Path
  2. 選擇 ipro_iot_tool.exe 位置

Q4: COM 埠連接失敗

問題：shake hand failed

解決方法：

  1. 確認 USB 線已連接且驅動正常
  2. 檢查 COM 埠號是否正確
  3. 關閉其他占用 COM 埠的程式
  4. 按下 RESET 按鈕後立即開始燒錄

Q5: 韌體檔案不存在

問題：韌體檔案不存在，請先建置專案

解決方法：

  1. 確認已執行 Configure 和 Build
  2. 檢查 apps/<project>/build/build_out/ 目錄
  3. 如果沒有檔案，重新執行 Build
